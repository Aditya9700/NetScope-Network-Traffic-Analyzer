NetScope System Architecture

NetScope is structured around a decoupled, modular design separating raw packet ingestion from telemetry visualization. Below is the sequence flow of packet data from the NIC (Network Interface Card) to the web browser.

1. Packet Ingestion

NIC captures raw Ethernet frames via Scapy raw socket bindings.

2. Header Parsing

Parser extracts layer parameters and byte payloads.

3. Thread-Safe Buffer

Packets append to concurrent deque and CSV logger with rotation.

4. REST API Gateway

Flask routes serve stats and JSON streams with security verification.

5. Canvas UI

Chart.js translates data science arrays into telemetry feeds.

Network & System Fundamentals

  • Layer 2/3/4 Decapsulation: Custom binary packet parsing mapping MAC addresses (Ethernet), Protocol Types (IPv4/IPv6), and Port numbers/Sequence keys (TCP/UDP/ICMP/ARP).
  • Multithreaded Race Prevention: Asynchronous thread creation running Scapy packet sniffers in the background while maintaining thread synchronization using `threading.Lock` across shared buffer variables.
  • High-Performance Data Science: Applying NumPy for packet distribution standard deviation and percentile calculations, and Pandas for time-series resampling.
  • System Log Integrity: Automatic CSV log rotations triggering backup procedures at 5MB limits to avoid disk leaks and maintain permanent trails.
  • OWASP Validation Controls: Magic byte validation on PCAP uploads preventing mime-type spoofing alongside secure-file limits.
